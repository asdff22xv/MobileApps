# Контроль служебных автомобилей (CarControl)

Мобильное приложение для мониторинга и учёта использования служебного автотранспорта.

## Функционал
- ✅ Добавление записей о поездках
- ✅ Учёт начального и конечного пробега
- ✅ Автоматический расчёт пройденного расстояния
- ✅ Отображение общего пробега автопарка
- ✅ Просмотр списка всех поездок
- ✅ Фиксация даты и времени поездки
- ✅ Информация о водителе и маршруте

## Технологии
- Язык: Kotlin
- Среда: Android Studio
- API: Минимальный Android 7.0 (API 24)
- Хранение данных: In-memory database (TripDatabase)
- UI: Material Design, RecyclerView

## Описание модулей

### Trip.kt - Модель данных
Содержит все поля поездки:
- carNumber - госномер автомобиля
- driverName - ФИО водителя
- startMileage/endMileage - начальный/конечный пробег
- destination - маршрут
- purpose - цель поездки
- distance - вычисляемое поле (пройденное расстояние)

### TripDatabase.kt - Хранилище данных
- Хранение всех поездок в памяти
- Добавление новой записи
- Получение списка всех поездок
- Подсчёт общего пробега

### CarAdapter.kt - Адаптер
- Отображение списка поездок в RecyclerView
- Форматирование даты и времени
- Отображение пробега в формате "начало → конец"

### MainActivity.kt - Главный экран
- Инициализация RecyclerView
- Отображение общего пробега
- Обработчик кнопки добавления
- Обновление UI при возврате с экрана добавления

### AddTripActivity.kt - Добавление поездки
- Форма с валидацией всех полей
- Проверка корректности пробега
- Сохранение в TripDatabase
- Возврат на главный экран

## Инструкция по запуску

### Способ 1: Открыть в Android Studio
1. Скачайте и установите Android Studio
2. Склонируйте репозиторий или скачайте ZIP
3. Откройте папку CarControl в Android Studio
4. Дождитесь синхронизации Gradle
5. Запустите на эмуляторе или устройстве (Shift+F10)

### Способ 2: Собрать APK
1. Откройте проект в Android Studio
2. Build → Build Bundle(s) / APK(s) → Build APK(s)
3. APK будет в папке `app/build/outputs/apk/debug/`
4. Установите на устройство

## Инструкция по использованию

### Добавление новой поездки
1. Нажмите кнопку "+" в правом нижнем углу
2. Заполните все поля:
   - **Госномер** - номер автомобиля (например: А123ВС777)
   - **Водитель** - ФИО водителя
   - **Начальный пробег** - показания одометра до поездки
   - **Конечный пробег** - показания после поездки
   - **Маршрут** - откуда и куда (например: Офис - Склад)
   - **Цель** - цель поездки
3. Нажмите "СОХРАНИТЬ ПОЕЗДКУ"

### Просмотр поездок
- Все поездки отображаются на главном экране
- Каждая карточка показывает:
  - Номер автомобиля
  - ФИО водителя
  - Маршрут
  - Пробег (начальный → конечный) и расстояние
  - Дату и время поездки
- Вверху отображается общий пробег всех автомобилей

## Примеры использования

### Пример 1: Учёт ежедневных поездок
- Водитель Иванов на авто А123ВС
- Утром: Офис → Склад (10 км)
- Днём: Склад → Офис (10 км)
- Общий пробег за день: 20 км

### Пример 2: Мониторинг расхода топлива
- Зная пробег и расход топлива можно рассчитать затраты
- Пробег за неделю: 500 км
- При расходе 10л/100км = 50 литров топлива

## Тестирование

### Успешные сценарии
✅ Добавление поездки со всеми заполненными полями
✅ Корректный расчёт пройденного расстояния
✅ Отображение общего пробега
✅ Проверка что конечный пробег больше начального
✅ Отображение пустого списка при первом запуске

### Обработка ошибок
❌ Попытка сохранить с пустыми полями - сообщение об ошибке
❌ Конечный пробег меньше начального - сообщение об ошибке
❌ Неверный формат чисел - сообщение об ошибке

## Известные ограничения
- Данные хранятся только в памяти (очищаются при перезапуске)
- Нет экспорта данных в Excel/PDF
- Нет авторизации пользователей
- Нет фильтрации по датам или автомобилям
